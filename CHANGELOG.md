## v0.5.0 - 2026-06-27

这一版是**稳定性 / 安全性集中大修 + 体验打磨**,含一次对整个代码库的系统排查与逐项修复。

### 新增

- **悬浮窗(托盘额度面板)** 仿 CodexBar:常驻置顶、点托盘图标切换显隐、长按窗口任意处即可拖动移位。
- **额度刷新进度条**:手动刷新额度时,右下角改为确定性进度条「正在加载额度 X / N」,多账号也能看着一格格涨满,不再干转圈、看着像卡死。
- **更新失败可重试**:自动更新的检查 / 安装失败后,弹窗新增「重试」入口(有待装更新则重试安装,否则重新检查)。
- 设备码登录遵守服务器返回的轮询间隔与 `slow_down` 退避,避免被限流误判。

### 改进

- **窗口控件**(左上角交通灯)打磨更精致:细内描边让浅底上边界清晰、悬停轻微放大、配色更柔和。
- **安全加固**:启用内容安全策略(CSP),收紧脚本 / 样式 / 连接来源;含管理密钥与 API key 的配置、OAuth token、`~/.codex/auth.json` 等凭据文件在 Unix 上收紧到 `0600`;`capabilities` 权限按实际所需收窄;下载的代理核心二进制增加 SHA256 完整性校验。
- **OAuth 更稳**:所有出站请求加连接 / 读取超时(网络半开不再永久卡住登录线程);token 交换遇上游瞬时 5xx / 断连改为让前端在有效期内重试,而非一次抖动就把登录钉死。
- **写上游代理地址**失败不再裸报「管理接口返回 HTTP 400」:版本不支持该运行时端点时静默降级(设置已存进 `config.yaml`、代理热重载会生效),地址不可达 / 无效时给出清晰提示。
- 透明无边框窗口在 macOS 上**真正透明**(修四角白直角)。

### 修复

- **核心锁中毒不再砖化整个 UI**:某个持锁线程一旦 panic,过去会让所有命令永久失败、界面卡死;现在统一自动恢复并清除毒化标记。
- **多账号刷新不再冻 UI**:把若干同步命令里的阻塞健康探测移出主线程。
- **日志「请求」tab 的删除**:此前点删除毫无反应(它清的是另一份「代理日志」,而非你看到的请求记录)。现在真正清空请求历史,并改为 **app 内二次确认 + 显示真实条数**,避免误删全部用量历史(它与仪表盘数据同源)。
- **仪表盘**:并发刷新时旧结果不再覆盖新结果、查询失败有明确反馈、自动刷新定时器不再被每次筛选打断。
- 第三方自定义服务商绑定密钥不再误报「密钥未绑 Codex」。
- 悬浮窗:浅色主题下不再透出桌面深色;拖动时不再被误判为失焦而自动隐藏。
- 其它:token 计数溢出保护、用量记录去重不再误删不同请求、agent 备份恢复增加路径校验防穿越等。

## v0.4.8 - 2026-06-23

### 新增

- 账号导入兼容 **`accounts-export.json`** 格式(`{ accounts: [{ token, refreshToken, email, accountId, … }] }`):自动把每个账号的扁平驼峰令牌转换成 CLIProxyAPI codex 账号(`token`→access、`refreshToken`→refresh、`accountId`→account_id),按 `codex-<邮箱>.json` 落地、支持一个文件多账号。该格式不带 `id_token`,导入时写空串占位,CLIProxyAPI 首次请求会用 `refresh_token` 续期补全。

### 改进

- 「API 密钥」页**防呆警告**:给密钥绑定了服务商、但运行环境缺少 `quotio-key-router` 插件时,弹横幅提示「绑定不会生效,代理仍按全局轮询命中所有池」。此前是悄无声息——看着绑了 Codex,请求却可能被轮询到别的池(用户实测踩坑)。
- 安装包资源打包改为**显式映射**(`bundle.resources` 不再用 `../` 回溯):避免 Tauri 把 `..` 转义成 `_up_/…` 导致内置 CLIProxyAPI 核心 / key-router 插件定位失败、首次运行误联网下载核心。让「装完即用、不联网」在 macOS 上也真正兑现(本机 Windows 已实锤资源从 `_up_/…` 改落到 `resources/proxy/…`)。
- 加固**管理 API 与 Cloudflare 隧道**流程:管理接口端点 URL 构造更稳(支持 HTTPS / 路径拼接);cloudflared 下载、解包(`.tgz`)、赋可执行更健壮;端口占用的进程检测与按端口杀进程更可靠。

### 修复

- 发版 CI:macOS / Linux 的 key-router 插件改为**下载失败即报错**(此前静默降级 `rm`,可能悄悄出一个不带插件、「按 key 路由」失效的包)。

## v0.4.7 - 2026-06-22

### 改进

- 启动方案行加回「**编辑**」按钮(在 `启动` 和 `删除` 之间):点开把该方案预填进表单,改完「保存方案」就按 id 更新那一条,不必删了重建。

### 修复

- 修复**停止 / 切换 Codex 后,上一套方案的 codex 进程没被杀掉、还拿着旧 key 在发请求**:此前「关 Codex」只按 `Codex.exe` 杀桌面应用 —— 用「终端」方案起的 codex(原生 CLI 二进制名可能带平台后缀、或是 node 起的)根本杀不掉,而终端又是用 `wt` 开的、记下的 pid 是个转瞬即逝的启动器并不可靠。结果切方案后旧 codex 还活着,日志里一直冒上一套方案的请求(典型:停了卫龙换成 codex,日志还在请求卫龙、会话卡思考)。现在改为按 `codex*` 进程名 + node 命令行一并收掉,停止 / 切换 / 启动自愈都生效。

## v0.4.4 - 2026-06-21

### 新增

- 智能体页 **Codex 多套启动方案**:一个 Codex 现在能保存多套启动配置,各自带 **账号 / 代理地址 / API 密钥 / 模型 / 思考程度**;一行一套、点哪套起哪套。**同一时刻只跑一套**——在另一套上点启动会先弹确认,再停掉当前那套并切过去。因为每套可填不同的代理地址 + 密钥,所以能**让不同方案走不同账号池**。升级后,原有的单套启动配置会自动迁移成第一套方案,配置不丢。
- 添加账号弹窗内置「**2FA 验证码**」面板:OAuth 登录需要二次验证码时,无需关闭弹窗或切换菜单(弹窗是全屏遮罩、且授权状态在弹窗内,切走会中断)——展开即可看到已保存 2FA 条目的实时验证码并一键复制(复用现有 2FA 金库)。

### 改进

- 智能体页卡片重整:Agent 名称去掉「CLI」后缀(`Codex CLI`→`Codex`、`Gemini CLI`→`Gemini`、`Amp CLI`→`Amp`),去掉冗余的英文描述行,按钮区更紧凑。
- Codex 一键启动更跟手:Codex 没在运行时不再白等固定 500ms;商店版应用启动把「拿 pid」的等待从最多 8 秒压到 best-effort 最多 1.5 秒——拿到就记、拿不到也不影响启动/停止(App 模式停止/监控本就按进程名兜底)。
- 智能体页:Codex 卡片去掉单独的「配置」面板(写永久 CLI 配置那套)——现在统一用「启动方案」配置 + 启动,免得和方案里的代理地址/密钥重复、混淆;「修复可见性」保留。其它 agent 的「配置」不变(那是它们唯一的接入方式)。
- Codex 启动方案加**密钥检查**:方案的 API 密钥没绑定到 Codex(绑到「全部服务商」或别的服务商)时,行内标「⚠ 密钥未绑 Codex」,点启动会先弹提示(这种密钥的 Codex 请求可能被路由到非 Codex 账号而报错),建议去「API 密钥」页绑到 Codex;「切换正在跑的方案」与「密钥未绑」合并成一个启动前确认。

### 修复

- 修复点「启动 / 停止 Codex」时窗口卡死「未响应」:`codex_start` / `codex_stop` 之前在 UI 主线程上同步跑完整段重活(关进程、写配置、修会话 sqlite、拉起进程,商店版还要轮询 pid 数秒),期间窗口完全无法响应。现改到后台 blocking 线程执行(与启动/停止代理一致),点击后只显示「启动中…」转圈,不再冻界面。
- 修复**升级后本地设置被重置**:此前只要某个设置字段的格式跨版本变了(典型是枚举值改名),整份 `settings.json` 就会因解析失败被重置成默认、并在启动时覆盖原文件。现改为**容错解析**——逐字段抢救,只回退放不进当前版本的那个字段,其余(代理 / 端口 / 语言 / 主题 / 调度规则等)原样保留。
- 智能调度「临近刷新优先」不再死磕快耗尽的账号:选中账号任一窗口余量 ≤ 3% 时视为耗尽,**提前踢出代理池、改用满血的待命账号**,等它窗口刷新(余量回 > 5%)再放回(3%~5% 为迟滞缓冲,避免边界横跳)。此前余量只剩 3% 但 API 仍 `allowed` 的账号会被一直选中、请求全部 429,而满血的待命账号闲置不用——现已修复。

## v0.4.3 - 2026-06-20

### 新增

- 账号导入兼容 **sub2api** 导出格式(`type: "sub2api-data"`):自动展开 `accounts[]`,把每个 OpenAI 账号 `credentials` 里的令牌转换成 CLIProxyAPI codex 账号格式(`chatgpt_account_id`→`account_id`、`expires_at`→`expired`,丢弃 codex 不需要的字段),按 `codex-<邮箱>.json` 落地、导入即用;支持一个文件多账号。其它平台暂跳过。

### 修复

- 修复服务商页账号重复显示（ChatGPT Team 等）：真正的原因是**大小写**——代理的 `/auth-files` 会把账号文件名转成小写返回，而本地账号目录保留原始大小写；前端把「代理账号」与「本地补充账号」合并时按大小写敏感比对，导致文件名含大写字母的账号（如 `codex-MartilloOlivia…`）被当成「本地独有」又列了一遍（纯小写文件名的账号则正常）。改为大小写不敏感比对；后端账号信息富化、智能调度重启账号时的同名匹配也一并修正（避免「待命」账号被误显示为「已禁用」）。
- 升级时自动刷新内置组件：kiro-rs sidecar 与「按 key 路由」插件原先只在首次安装时落地、升级不会替换，旧机子可能一直用着旧的 sidecar/插件。现在改为与安装包内置版本按文件大小比对，不一致即覆盖管理目录的旧版本——与核心（CLIProxyAPI 一直是每次启动覆盖）保持一致，三个组件升级都会跟着更新。

## v0.4.2 - 2026-06-20

### 新增

- 安装包预置 CLIProxyAPI v7.2.20 核心（Windows / macOS / Linux 三端各自打包），全新安装即用，首次运行不再联网下载核心。
- 安装包内置 kiro-rs sidecar 与「按 key 路由」插件：由 CI 从仓库内 vendor 源码在 **Windows / macOS / Linux 三端各自编译**并随包分发，Kiro 代理池与 key→池隔离全平台离线即用、无需手动放置（二进制可从内置资源自动装载到管理目录）。便携版（Windows）同样内置。
- 2FA 管理新增「名称/邮箱」输入框，可一次录入「邮箱 + 密钥」直接保存，便于按账号成批管理。
- 新增请求错误日志：失败请求（HTTP ≥ 400）写入本地 `errors-YYYY-MM-DD.jsonl`（含原始用量快照），自动保留 30 天；设置页新增「打开日志目录」按钮，方便取证排查。

### 改进

- 2FA 页面样式全面对齐应用主题：主色、暗色模式、圆角/边框统一改用设计 token；验证码与倒计时改为并排、倒计时改用琥珀色；已保存列表中邮箱与密钥中间打码，避免敏感信息直接暴露。

### 修复

- 修复 ChatGPT Team 账号导入：同一工作区的多个成员共享同一 `account_id`，之前仅按 `account_id` 去重会把成员当成同一账号误删，现按 `account_id + email` 区分，不同成员各自保留。
- 修复主窗口启动卡顿：`get_app_state` 改为异步 + 后台线程执行，启动时的代理健康探测与 auth 文件读取不再阻塞 UI 主线程。
