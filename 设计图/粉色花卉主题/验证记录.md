## 本轮设计稿对齐修订

- 修复玫瑰主题把 app-shell 从固定窗口改成内容撑高的问题。主窗口重新严格占满视口，内层页面滚动不再导致整页高度变成 1200px 以上。
- 仪表盘人物 Hero 改为 `.usage-dash` 的直接装饰层，不再受滚动区顶部裁切；宽屏和 1081–1599px 均保留完整半身构图，≤1080px 才隐藏。
- 原 `character-hero.webp` 的服装、姿势和大面积留白与设计稿不符；初版 `character-dashboard-hero-v2.png` 又是无透明通道的 RGB 原图，矩形背景会压住人物两侧的花卉。现改用 1586×992 RGBA 透明抠图 `apps/desktop/public/rose/character-dashboard-hero-cutout-v4.png`，人物身份参考现有头像，服装和手势参考仪表盘设计稿；花卉、花瓣、背景和金色纹样继续复用现有素材。
- 仪表盘筛选区去除继承产生的额外 flex 间距，KPI、图表和账号汇总表重新按设计稿纵向落位。
- 仪表盘账号表压缩到 28px 行高；1729×910 下可在同一首屏完整显示 5 行账号和分页。
- 仪表盘 1729×910 下取消了 10px 外层底部占位，滚动区 `clientHeight=834`、`scrollHeight=834`；隐藏滚动条轨道，窄窗口仍可用滚轮滚动。
- 仪表盘左下花卉保留为 `.usage-dash::after` 的 fixed 背景，右下花卉拆为独立的镜像裁剪背景 `.rose-dashboard-bottom-flower--right`；两者均位于 `z-index: 1`，卡片所在 `.dashboard-scroll` 为 `z-index: 3`。1729×910 浏览器视觉确认右侧竖枝贴住右下边缘、主花簇向左展开，并裁掉远处散落花瓣；装饰不参与布局且不会遮挡卡片内容。
- 侧栏底部花卉装饰改为固定在窗口层，不再参与侧栏滚动高度计算；1180×760 下侧栏 clientHeight 与 scrollHeight 均为 760px。
- 智能体页将已接入列表改成内部滚动，只展示首屏所需高度；运行概览去除不属于设计稿的路线图，启动方案与发现区进入同一首屏。
- 关于页保留 Q 标识，增加独立的大型人物半身列；人物不再错误替换品牌标识。
- 2FA 新增仅在 rose 主题启用的专用双栏结构：左侧账号列表、搜索、导入和保险库概览，右侧验证码、账号信息、密钥和备份恢复；原有查询、保存、QR、导入导出、编辑删除和 TOTP 逻辑均复用。
- 2FA 没有新增“已加密”或“已创建备份”等无法由现有数据证明的文案，仅描述本地存储和手动导出。
- 2FA 只在宽度 ≤1080px 改为单栏；默认 1180×760 仍保持双栏，并由左右面板各自滚动。
- 额度页新增独立的 quota-rose.css，所有规则均限定在 .quota-redesign，未修改菜单栏/侧栏。调度横幅、四项汇总、服务商总面板和账号卡片按 1746×901 设计稿重新建立尺寸与层级，不再只是颜色换肤。
- 额度页右上花簇继续复用 rose-corner.png，按用户最新裁图重新校准为 420×280 的完整旋转构图（top -75/right -15），不再裁掉下半玫瑰；同时用 CSS 补齐四组金色四角星芒。头像复用 character-avatar.png，服务商标题改用粉色遮罩版 rose-mark.svg。现有素材完整，本页未调用 imagegen 生成重复素材。
- 额度页调度主题说明仅在 Rose 主题显示，浅色/深色保持原有结构；中英文均补充对应文案。自定义服务商选中时不再错误借用首个内置服务商账号。
- 额度页宽屏汇总列按设计稿非等宽分隔精确落位；账号面板 Header 底线和首行卡片统一落到 y=372，首卡 rank 落到 x=263/y=389，第六张单卡保留约 382×224 的独立尺寸。
