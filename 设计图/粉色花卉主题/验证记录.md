# 粉色花卉主题验证记录

验证日期：2026-07-17

## 实现范围

- 新增可持久化的 `rose` 主题模式，前端 TypeScript 与 Rust Serde 值保持一致。
- 主窗口与悬浮窗共用主题解析和应用逻辑，并通过 `settings-changed` 事件即时同步。
- 设置页新增“粉色花卉 / Rose Floral”选项。
- 为仪表盘、额度、服务商、智能体、API 密钥、日志、设置、2FA、关于页、全局弹窗和悬浮窗提供主题样式。
- 正式运行时素材均从 `apps/desktop/public/rose/` 引用，未把页面效果图作为程序背景。


## 设计稿对齐修订

- 删除了错误的独立空白 Hero 横幅，人物、右上花簇、花瓣、双侧枝叶和手写文案改为与标题/筛选区共用的装饰层。
- 侧栏按设计稿收窄为 204px，并将头像信息卡改为玫瑰标识与手写体 Quotio 字标。
- 仪表盘的筛选控件、KPI、面板圆角、图表高度和纵向间距已按参考图压缩，粉色主题趋势图改为粉色曲线。
- 七个云纹素材均按 `cloud-assets.md` 映射：页面角纹、分段括号、人物枝叶、额度/关于九宫格框、智能体卡片回勾纹、日志分界云结和人物花环。
- 智能体页增加右上人物花环，关于页增加人物两侧镜像枝叶；装饰层不会覆盖按钮、焦点环或弹窗。
- 宽度低于 1600px 时隐藏大型人物 Hero，避免动态筛选器换行后发生遮挡；宽屏保持设计稿构图。
## 自动化验证

- `npm --prefix apps/desktop test -- --run`：5 个测试文件，22 个测试全部通过。
- `npm --prefix apps/desktop run build`：通过；仅保留项目已有的动态导入和大 chunk 警告。
- `cargo test -p quotio-types`：6 个测试全部通过。
- `git diff --check`：通过；仅有 Windows 工作区的 LF → CRLF 提示。
- `apps/desktop/public/rose/manifest.json` 中列出的 14 个素材均存在，代码引用路径均有效。

## 静态质量检查

- 主题主按钮使用较深粉色并保留白字，避免被普通白色按钮规则覆盖。
- 根元素与 `body` 在玫瑰主题下保持透明，悬浮窗圆角外不会被实色画布填充。
- 当前导航项仅由按钮容器绘制高亮，图标和文字不再重复绘制背景。
- 弱化文字对比度提升至至少 4.5:1；键盘焦点环使用实色主色。
- 花卉与人物装饰均设置为非交互层，不覆盖表格、OTP、二维码或表单控件。
- 未使用固定背景、大面积模糊或多层 `backdrop-filter`，避免 WebView2 性能回退。

## 浏览器验收限制

已尝试使用应用内浏览器打开 `http://127.0.0.1:4173/` 做 960×640 和 360px 视觉验收，但企业网络策略禁止浏览器工具访问本机回环地址，同时禁止改用其他浏览器通道绕过。因此本次没有生成自动截图，也没有完成逐页点击式浏览器验收。

建议发布前在实际 Tauri 窗口中快速检查：主题切换、设置下拉浮层、主窗口各页面滚动、弹窗/2FA 对比度，以及 360px 悬浮窗圆角和横向溢出。
