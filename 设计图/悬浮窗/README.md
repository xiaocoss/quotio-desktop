# Quotio 悬浮窗设计交付

本目录只包含 Quotio 窄长桌面悬浮窗的设计交付，不修改、替换或约束现有业务代码。界面保留当前功能与数据语义，目标是为后续 React、CSS 与 SVG 实现提供一致的视觉基准。

## 文件说明

最终完整交付包含以下文件：

- `floating-window-redesign.png`：竖向高分辨率主效果图，完整展示单个窄长悬浮窗。
- `floating-window-reference.html`：可直接在浏览器打开的单文件静态实现参考，不以 PNG 作为界面背景。
- `design-tokens.json`：逻辑画布、颜色、字体、间距、圆角、阴影和组件尺寸契约。
- `imagegen-prompt.md`：Image 2 使用的结构化提示词、参考图角色、约束与避免项。
- `README.md`：交付索引、开发建议、关键尺寸与视觉方向。
- `assets/floating-window-bg.svg`：暖米白纸感底色与克制的蓝色、薄荷绿色柔光。
- `assets/card-noise.svg`：卡片表面的低透明度轻噪点纹理。
- `assets/floating-window-icons.svg`：停止、刷新、打开、退出、时钟、日历和状态图标集合。
- `assets/quota-indicators.svg`：健康点阵与健康、预警、危险额度条的装饰性视觉参考。

## 开发落地建议

- 窗口建议使用三行 CSS Grid：顶部控制区、中部 `minmax(0, 1fr)` 滚动区、底部 `144px` 操作区；仅账号列表设置 `overflow-y: auto`。
- 顶部的 `Quotio`、本地地址、停止按钮及工具标签始终可见；底部三行操作固定在窗口底部，账号卡在二者之间独立滚动。
- 邮箱、健康百分比、PLUS、到期时间、剩余时间、Session 与 Weekly 均绑定真实业务数据；长邮箱单行省略，并通过 `title` 或等效无障碍文本暴露完整值。
- 动态健康点阵必须按真实状态数据生成 DOM 节点并由 CSS 着色；动态进度条必须按真实百分比设置 CSS 宽度。`assets/quota-indicators.svg` 只用于视觉参考，不应承载运行时数据。
- 状态不能只靠颜色表达：绿色、橙色和红色进度旁始终保留百分比文字，预警与危险状态可配合浅色背景或状态文案。
- 图标优先复用本地 SVG sprite，并以 `currentColor` 继承状态色；不要引入外部字体、图片服务或运行时网络依赖。

## 关键尺寸

| 项目 | 规格 |
| --- | --- |
| 逻辑窗口 | `360 × 960px` |
| 主效果图建议 | `720 × 1920px` 或更高，窗口本体约 `3:8` |
| 窗口圆角 | `18px` |
| 内容水平内边距 | `12px` |
| 账号卡高度 | `156px` |
| 账号卡间距 | `10–12px` |
| 账号卡圆角 | `14px` |
| Session / Weekly 进度条 | `6px` 高，全圆角 |
| 健康点 | `7px` 直径，建议 `4px` 间距 |
| 滚动条 | `4px` 宽，低对比圆角滑块 |
| 底部操作 | 3 行，每行 `48px`，总高 `144px` |

## 视觉方向

- 保留单个窄长、完整边框的桌面悬浮窗，不使用手机外壳、桌面场景或营销页构图。
- 以暖米白 `#F6F3EC` 为页面底，白色 `#FFFFFF` 为主要表面，使用细冷灰边框和短而轻的阴影建立层级。
- 主操作与当前工具标签使用蓝色 `#3978F6`；健康、预警、危险分别使用绿色 `#35B96B`、橙色 `#F59E0B`、红色 `#EF4444`。
- 字体清晰、紧凑且易扫读，优先 HarmonyOS Sans SC、MiSans、PingFang SC、Inter 与系统无衬线字体；本地地址可使用等宽字体。
- 装饰必须能用普通 CSS/SVG 落地，避免暗色主题、重玻璃模糊、霓虹、3D、复杂材质和无功能意义的额外控件。
