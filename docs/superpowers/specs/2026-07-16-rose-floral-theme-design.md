# Quotio 粉色花卉可切换主题设计规格

## 目标

在不改变 Quotio 现有业务逻辑和默认主题的前提下，新增一套可切换的粉色花卉人物主题。主题参考用户提供的界面截图，使用奶白、浅粉、玫瑰红和淡金色，结合玫瑰、花瓣、柔光、半透明圆角卡片与原创人物视觉，形成统一的限定版体验。

## 范围

覆盖现有设计稿和应用中的以下界面：

- 仪表盘
- 额度
- 服务商
- 智能体
- 日志与代理日志
- 设置
- 2FA
- 悬浮窗
- 关于

现有 `设计图` 目录中的旧设计交付将完整归档到 `设计图/原版/`。新设计稿放入 `设计图/粉色花卉主题/`，按页面建立独立目录。归档仅调整设计资料位置，不删除任何 PNG、HTML、设计 token、提示词或 SVG 素材。

## 视觉系统

### 色彩

- 页面底色：暖奶白与极浅粉渐变。
- 主色：玫瑰粉和低饱和莓红，用于主要按钮、选中状态和关键数据。
- 辅色：淡金色，用于星光、边缘高光和少量强调。
- 状态色：保留成功、警告、错误、信息的语义差异，但降低饱和度以融入主题。
- 深色文字：使用暖黑或深棕红，避免纯黑造成生硬对比。

### 组件

- 卡片使用半透明奶白背景、细粉色描边、柔和阴影和较大圆角。
- 侧边栏使用浅色布局，选中项以粉色胶囊背景和玫瑰色图标区分。
- 表格、图表、输入框、对话框和通知保持现有信息层级，只替换视觉 token 与装饰层。
- 花卉和人物图像不承载可交互文字；所有文字、数据和控件继续由 React、CSS、SVG 与 Recharts 渲染。

### 装饰控制

玫瑰、枝叶、花瓣和星光集中在页面边缘、标题区及空白区域，不能遮挡数据或交互控件。高密度数据页减少人物占比，以花卉角饰和纹理为主；仪表盘和关于页可以使用更完整的人物主视觉。

## 人物与图像资产

使用 Image 2 生成原创虚构人物，不复制参考截图中真人的具体身份。角色保持统一的面部特征、深色双马尾或长卷发、粉色服装体系、柔和写真光线和玫瑰主题。

图像资产与整页设计图分开交付：

- 独立方形头像，供侧边栏、账户区和关于页引用。
- 独立半身或全身人物素材，供仪表盘等主视觉区域引用。
- 页面背景、玫瑰角饰、花瓣和纹理素材。
- 各页面高保真设计效果图，仅用于视觉参考，不直接作为应用界面背景。

同一人物可在不同页面使用不同姿势，但面容、发型、服装语言和色彩必须保持一致。生成结果需检查人物一致性、异常手指、裁切、文字伪影和水印。程序使用的图片保存到应用公共资源目录，设计交付副本保存到新主题设计目录。

## 应用架构

采用主题覆盖层方案，不复制业务页面组件。

- 在根节点使用稳定的主题标识，例如 `data-theme="rose"`。
- 把可复用颜色、阴影、圆角、背景和边框抽象为 CSS variables。
- 新主题通过主题变量和限定选择器覆盖现有页面样式。
- 页面可按需引用主题专属装饰图片，但不得分叉数据获取、状态管理或命令调用逻辑。
- 设置页新增主题选择项，至少包含现有默认主题与粉色花卉主题。
- 用户选择持久化，并在应用启动时尽早恢复，避免明显的主题闪烁。
- 悬浮窗与主窗口共享主题选择；若技术边界要求独立读取持久化值，也必须呈现同一选择结果。

## 页面设计顺序

逐页设计和验证，顺序如下：

1. 仪表盘：建立主题的整体视觉基准和人物主视觉。
2. 额度：验证数据卡片、进度条和密集信息表达。
3. 服务商：验证品牌标识、连接状态和操作区。
4. 智能体：验证流程图、路由关系和人物装饰共存。
5. 日志与代理日志：验证表格、筛选器、状态色和高密度数据。
6. 设置：加入主题选择控件并验证表单组件。
7. 2FA：验证安全信息、二维码和倒计时状态。
8. 悬浮窗：把主题压缩到小尺寸窗口。
9. 关于：强化人物、品牌与限定版视觉收尾。

每张效果图确认可由现有前端技术实现后，再作为对应页面的开发参考。

## 数据流与持久化

主题切换只影响展示层：

1. 设置页更新主题偏好。
2. 偏好写入现有适合的本地持久化机制。
3. 应用根组件读取偏好并设置根主题标识。
4. CSS variables、页面覆盖样式和主题资产根据该标识生效。
5. 所有额度、日志、服务商、智能体和设置数据继续走现有数据流。

若保存值无效或读取失败，应用回退到当前默认主题，不影响启动和业务操作。

## 验证标准

- 默认主题在视觉和功能上不发生非预期变化。
- 粉色花卉主题可从设置页切换，重启后仍保持选择。
- 所有页面、弹窗、空状态、加载状态、错误状态和悬浮窗均有完整主题覆盖。
- 头像是独立文件，并由程序资源路径引用。
- 人物主视觉和装饰素材不包含不可控界面文字或水印。
- 主题在应用最低尺寸 `960 × 640` 下不遮挡关键控件，也不产生横向溢出。
- TypeScript 构建和现有测试通过；新增主题持久化与映射逻辑具有针对性测试。
- 逐页进行运行截图检查，并与批准的设计图对照。

## 非目标

- 不修改额度、日志、代理、服务商、智能体或 2FA 的业务行为。
- 不为粉色主题复制整套 React 页面组件。
- 不直接复刻参考图中的真人身份或 Codex 产品界面。
- 不把生图中的文字当作正式界面内容。
- 不在本次工作中新增与主题无关的功能或大规模重构。
