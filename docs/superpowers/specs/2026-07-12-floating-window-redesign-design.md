# Quotio 悬浮窗重设计规格

## 目标

基于现有窄长悬浮窗截图，制作一套与 Quotio 其他浅色设计稿一致、可由 React/CSS/SVG 落地的高保真设计交付包。保留现有功能和数据语义，只重构视觉层级、间距、状态表达和滚动体验。

最终交付位于 `设计图/悬浮窗/`，不修改桌面应用业务代码。

## 已确认方向

采用方案 A：紧凑型桌面工具面板。

- 保留窄长悬浮窗形态，不改造成完整主窗口或手机界面。
- 使用暖米白页面底、白色半透明分层面板、深灰文字、蓝色主操作、绿色健康状态、橙色额度预警和红色危险操作。
- 信息密度保持较高，但通过固定头部、账号卡片分组、清晰进度条和固定底部操作区提升可扫读性。
- 与现有仪表盘、额度、智能体和设置设计包保持同一套圆角、细边框、柔和阴影与轻纸感语言。

## 画布与整体结构

- 主效果图采用约 `720 × 1920` 的 3:8 竖向画布，以便 Image 2 清晰生成文字和细节；界面本身按 `360 × 960` 的逻辑尺寸设计。
- 外窗圆角约 `18px`，边框为低对比冷灰色，使用克制的悬浮阴影。
- 页面分为三个稳定区域：固定顶部控制区、可滚动账号区、固定底部操作区。
- 顶部和底部之间的账号区独立滚动；滚动条细、低对比，但应清晰表示当前位置。

## 组件设计

### 顶部控制区

- 第一行左侧显示 `Quotio` 品牌文字，中间显示本地地址 `http://127.0.0.1:28317`，右侧保留红色 `停止` 按钮。
- 地址使用等宽或近似等宽的小字号，并允许视觉省略；停止按钮保持明确的危险语义，但面积不过大。
- 第二部分为工具筛选标签：`Codex`、`Claude`、`Copilot`、`Antigravity`、`Kiro`、`GLM`、`Trae`、`cursor`。
- 当前工具 `Codex` 使用蓝色浅底与蓝色文字；其余标签为低对比文字按钮。标签允许自然换行，不使用横向滚动。

### 账号滚动列表

- 每个账号使用紧凑卡片或分组行，卡片之间以 `10–12px` 间距分隔，避免原图中连续文本堆叠。
- 账号标题行包含截断邮箱、健康百分比和 `PLUS` 方案标签。健康百分比按状态使用绿色、橙色或红色。
- 元信息行保留重置次数、到期日期和剩余时间；使用图标、标签与弱化文字建立层级。
- 健康点阵位于元信息与进度条之间，默认显示最近状态轨迹；成功点为绿色，普通点为浅灰，异常点为橙色或红色。
- `Session` 与 `Weekly` 各占一行：左侧名称，中部或右侧显示剩余时间，最右显示百分比；下一行是 `6px` 高进度条。
- 进度颜色规则：健康为绿色，预警为橙色，危险为红色。不能只依赖颜色，百分比文本必须始终可见。
- 单个账号卡片逻辑高度约 `150–168px`，一个视口内可完整看到约 4 个账号并露出下一项，明确列表仍可滚动。

### 底部操作区

- 固定在窗口底部，使用顶部细分隔线与轻微半透明背景。
- 包含 `刷新`、`打开 Quotio`、`退出 Quotio` 三个纵向操作行。
- `刷新` 为普通操作，`打开 Quotio` 使用蓝色图标或文字强调，`退出 Quotio` 使用克制的红色危险态；整行均为可点击区域。
- 每行高度约 `48px`，图标和文字左对齐，保证窄窗下仍易点击。

## 状态与交互语义

- 正常账号：绿色健康百分比、绿色额度条。
- 额度预警：橙色健康或额度值，并使用浅橙背景提示，不改变卡片结构。
- 严重不足或异常：红色进度条和红色数值；保留完整文字，避免只用警告图标。
- 当前工具标签与停止按钮必须在任何滚动位置可见。
- 账号邮箱超长时使用单行省略；HTML 参考稿通过 `title` 暴露完整值。
- 静态参考稿使用示例数据模拟状态，不接入真实接口，也不新增业务字段。

## 视觉令牌方向

- 页面底色：`#F6F3EC`；强表面：`#FFFFFF`；柔和表面：`#F7F9FC`。
- 主文字：`#1F2937`；次文字：`#667085`；弱文字：`#98A2B3`。
- 主色：`#3978F6`；成功：`#35B96B`；预警：`#F59E0B`；危险：`#EF4444`；方案标签使用蓝紫渐变或纯蓝紫色。
- 卡片圆角约 `14px`；控件圆角约 `10px`；胶囊标签为全圆角。
- 字体优先使用 `HarmonyOS Sans SC`、`MiSans`、`PingFang SC`、`Inter` 与系统无衬线字体。
- 阴影应轻且短，主要依赖边框和表面层级，不使用重玻璃模糊或霓虹光效。

## 交付文件

`设计图/悬浮窗/` 包含：

- `floating-window-redesign.png`：使用 Image 2 生图能力生成并验证的高保真主效果图。
- `floating-window-reference.html`：单文件、可直接打开的静态参考实现，不把 PNG 当背景。
- `design-tokens.json`：画布、颜色、字体、间距、圆角、阴影和专属组件尺寸。
- `imagegen-prompt.md`：最终生图提示词、参考图角色、约束及避免项。
- `README.md`：文件说明、开发落地建议、关键尺寸和视觉方向。
- `assets/floating-window-bg.svg`：暖色纸感与冷色柔光背景。
- `assets/card-noise.svg`：卡片表面轻微噪点。
- `assets/floating-window-icons.svg`：头部、状态与底部操作图标集合。
- `assets/quota-indicators.svg`：健康点阵和额度进度视觉参考。

## Image 2 生成要求

- 原截图作为结构和内容语义参考，不要求逐像素复刻。
- 必须保持单个窄长桌面悬浮窗完整入镜，不生成手机外壳、桌面场景、营销海报或多个窗口。
- 中文与英文界面文字应尽量清晰；邮箱可使用示例或遮罩文本，但组件层级、百分比和标签必须可信。
- 重点保证顶部固定控制区、至少四个完整账号卡片、可见滚动提示和底部三个操作行。
- 禁止暗色模式、强霓虹、过度渐变、3D 插画、品牌 Logo、水印和无法用普通 CSS/SVG 实现的装饰。

## 验证标准

- 主效果图完整呈现悬浮窗，没有裁切窗口边框或底部操作区。
- 原截图中的主要功能和数据类型全部保留。
- 账号标题、状态、元信息、健康点阵、Session 与 Weekly 层级一眼可区分。
- PNG、HTML、设计令牌和 SVG 素材采用同一套颜色与尺寸逻辑。
- HTML 在本地直接打开后，无外部资源依赖、无横向溢出，固定头尾和中部滚动结构可用。
- 所有交付物仅新增到设计目录，不改动现有业务代码。
