# Quotio v0.6.4 免安装版重新打包设计

## 目标

使用仓库中现有的 `target/release/quotio-desktop.exe` 重新组装 Windows x64 免安装版本，不重新编译源码，也不把当前工作区尚未提交的修改带入主程序。

最终交付物为：

- `dist-portable/Quotio_0.6.4_x64_portable/`
- `dist-portable/Quotio_0.6.4_x64_portable.zip`

## 方案选择

采用仓库现有的 `npm run package:portable` 流程。该命令调用 `scripts/package-portable.ps1`，读取 Tauri 配置中的 `0.6.4` 版本号，以已编译主程序为输入，重新创建便携目录和 ZIP。

不直接复用旧 ZIP，因为本次目标是明确重新组装并验证产物；不生成仅含单个 EXE 的精简包，因为那会遗漏离线代理组件；不运行 `npm run release`，因为它会重新编译源码并额外生成安装包，与已确认的范围不符。

## 打包内容

便携目录包含：

1. `Quotio.exe`：复制自 `target/release/quotio-desktop.exe`。
2. `resources/icon.ico`：使用现有 Tauri 构建资源中的图标。
3. `resources/proxy/windows/CLIProxyAPI.exe`：从 `%APPDATA%/Quotio/proxy/` 复制，使代理功能可离线启动。
4. `resources/kiro/windows/kiro-rs.exe`：从 `%APPDATA%/Quotio/kiro-rs/` 复制。
5. `resources/proxy/windows/plugins/quotio-key-router.dll`：从 `%APPDATA%/Quotio/proxy/plugins/` 复制。
6. 仓库中各平台代理资源的说明文件。

当前机器上的三个离线组件均已确认存在。

## 数据与兼容性边界

该产物是“解压即用、无需安装”的版本，但不是把所有运行数据都写入程序目录的完全便携模式。应用配置仍写入 `%APPDATA%/Quotio`，缓存仍使用 `%LOCALAPPDATA%/Quotio`。Windows 机器还需要可用的 Microsoft Edge WebView2 Runtime。

## 错误处理

- 若现有主程序缺失，打包脚本必须失败并提示先构建，不能生成不完整 ZIP。
- 打包目录和同名 ZIP 在组装前删除并重新创建，避免旧文件残留。
- 若某个离线组件缺失，脚本会给出警告；本次验证要求三个组件都出现在最终 ZIP 中。

## 验证

完成组装后执行以下检查：

1. 确认便携目录和 ZIP 均成功生成且大小非零。
2. 检查 ZIP 内含 `Quotio.exe`、`CLIProxyAPI.exe`、`kiro-rs.exe` 和 `quotio-key-router.dll`。
3. 比较 ZIP 解包目录中的 `Quotio.exe` 与 `target/release/quotio-desktop.exe` 的 SHA-256，二者必须一致。
4. 记录最终 ZIP 的文件大小、修改时间和 SHA-256，供交付核对。

本次操作不修改应用源码、版本配置或用户现有未提交改动。
